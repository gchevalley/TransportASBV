Procedures de sauvegarde 

Lorsque vous exécutez l'application de gestion de transport le système crée au démarrage  une sauvegarde totale qui peut être utilisée en tout temps pour reconstruire votre plateforme applicative. La sauvegarde crée un fichier de type ZIP file qui est automatiquement envoyé a l'adresse mail du compte  gmail de l'ASBV de même qu'a l'administrateur du système. Vous pouvez a volonté créer une sauvegarde en utilisant la fonction sauvegarde de la page d'accueil. Le nom du  fichier de sauvegarde contient la date et l'heure (  Exemple  : backup-2011-01-14 17-03-18.zip ). Cette fonction n'est disponible que lorsque vous êtes identifié et reconnu comme administrateur de filiale  ( Voir le profil du transporteur )

Ecran pour la sauvegarde  :

Des que la demande de backup est activee le syteme creera automatiquement le  ZIP file de sauvetage contenant  toutes les donnees  pour reconstruire l'application. Une copie de cette sauvegarde sera envoyee a l'administrateur de la filiale et une autre copie dans le compte Email de la filiale.

Une autre fonction permet de  restaurer la partie applicative et la base de données des transports a partir d'un fichier ZIP crée par la fonction sauvegarde. Cette action va automatiquement mettre a jour les données et le code de l'application tel qu'il était  a  l'heure de la création de la sauvegarde.  Toutes les données modifiées depuis cette  date seront  perdues.

Ecran pour la restauration de l'application :

Utiliser l'option Browse pour chercher sur votre plateforme le  fichier ZIP contenant les données de sauvegarde de l'application et  cliquez sur soumettre pour l'exécution de cette tâche.  Soyez patients et attendez le message annonçant la fin de l'opération.
