Directory structure  in c:/ASVB/TransportASVB
*********************************************

•	le dossier admin contient les différentes classes.
o	le principe est toujours le même, si un id existant dans la DB est passé au constructeur l'objet est monté à l'aide des données de la base, sinon une nouvelle entrée est crée.
o	glm est le modèle de calcul/ranking. Il se base sur les distances entre le point départ et le domicile du transporteur potentiel, les habitudes entre passager-transporteur, la distance parcourue ainsi que le nombre de trajets depuis le début de mois
o	un bénévole peut faire partie de plusieurs filiales avec des droits différents. Vous trouverez donc un super id et un id de filiale. Des fonctions statiques ont été programmées pour passer facilement de l'un à l'autre.
•	le dossier website contient le site web et le sous-dossier processing qui effectue les traitements selon le module (transporteur, bénévole, passager, filiale etc) et l'action (ajouter, retirer une disponibilité etc.) concernés en faisant appel aux fonctions des différentes classes.
•	chaque lien, formulaire déclenchent le rechargement d'index.php qui va s'occuper de rediriger et de charger le contenu approprié à l'aide des modules et des actions possibles, par exemple : module=benevole action=list.
•	le code xHTML est produit à l'aide de fonctions statiques directement implémentées dans les différents classes sous le nom form_xxx.
